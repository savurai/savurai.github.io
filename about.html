About This Project

This project demonstrates an end-to-end object detection pipeline that uses Roboflow for dataset management and model hosting, and a lightweight browser frontend to run live detections from your webcam.

How it works

  • Dataset & Training: Images were annotated and augmented using Roboflow, then trained with a YOLO-family detector on Roboflow cloud GPUs.
  • Model Hosting: The trained model is published via Roboflow Serverless/Hosted Inference API for easy integration into web or mobile apps.
  • Frontend: A simple HTML/CSS/JS UI captures camera frames, sends them to Roboflow for inference, and visualizes detections with an HTML5 canvas.

Key tools & links

  • Roboflow — dataset management, augmentation, cloud training, hosted inference
  • YOLO (family) — popular object detection architectures used in industry
  • getUserMedia / WebRTC — webcam capture in browser
  • Canvas API — drawing overlays and labels

Dataset & Classes

The model recognizes common electronic components used in lab and prototyping workflows:

  • Resistors — passive components that oppose current
  • Pulse generators — devices that produce pulses for testing circuits
  • Microprocessors — CPU units found on boards
  • Transistors — active semiconductor devices used as switches/amplifiers
  • LED — light emitting diodes
  • Capacitors — energy storage passive components
  • Heatsink — thermal dissipation hardware
